## v0.3.0
### Added
- Add admin settings controls and GraphQL support for removing configured external labeler subscription URLs.
- Add a read-only post-deploy API smoke test suite so operators can verify public GraphQL health, schema, records, and pagination after deployment.
- Add an opt-in API smoke write-through check so operators can verify ATProto create, update, and delete ingestion for Certified profiles and Hypercert activity claims.
- Add the official GraphiQL Explorer plugin so users can build public and admin GraphQL queries by checking and unchecking schema fields directly in the GraphiQL sidebar.
- Add presence-only GraphQL filters for complex top-level lexicon fields and make malformed typed union values resolve to null instead of surfacing GraphQL errors.
### Changed
- Add first-class external ATProto labeler support across ingestion and public GraphQL. Hyperindex can now subscribe to configured com.atproto.label.subscribeLabels streams, persist raw external label events with resumable cursors, expose active labels through subject lookups and record-level externalLabels fields, and filter generated record queries by external labels before pagination so clients can page through labeled records reliably. This also adds database indexes, safer subscription URL validation and log redaction, and documentation for running the labeler pipeline with Tap.
- Fail readiness checks and expose diagnostics when an external labeler subscription reaches a non-retryable cursor error, so operators can reset stuck labeler ingestion instead of serving stale label data silently.
- Split liveness and readiness probes so `/health` stays process-only while `/ready` reports database and configured labeler diagnostics failures.
- Increase the GraphQL connection page-size cap from 100 to 1000 records per request.
- Reduce backend Railway deploy overhead by pruning the backend Docker build context, using the matching Go builder image, and stripping release binary debug metadata.
- Embed the backend build version during Makefile and Docker builds so deployed status endpoints report the released Hyperindex version.
### Fixed
- Fix record-level external label queries so CID-scoped records can hydrate labels without PostgreSQL rejecting the internal lookup key.

## v0.2.0
### Added
- Add a Lexicons page ZIP upload flow so admins can install unpublished lexicon JSON archives without returning to onboarding.
- Add frontend Railway deployment configuration so the client service uses its root page as the health check with a longer startup timeout.
### Breaking
- Rename Hypergoat technical identifiers to Hyperindex, including the Go module path, command entrypoint, binary names, Docker/Compose defaults, and session cookie name.
### Changed
- Route port redirects through the proxy so deployments consistently expose the expected service endpoint.
### Fixed
- Prevent active accounts from being purged during Tap identity processing so indexed records are not removed after harmless identity updates.

- user

- operator
## v0.1.0
### Added
- Automate release-note batching so maintainers can generate a reviewable release PR from GitHub Actions on main.
- Add a curated release-note workflow so noteworthy changes can be captured in a consistent, reviewable format.
### Changed
- Change admin access management so ADMIN_DIDS on the backend is the source of truth, removing in-app admin DID mutation paths and showing the admin list as read-only in onboarding and settings.
- Update the release workflow so rerunning it after the release PR is merged creates the version tag and publishes the matching GitHub Release from the generated Changie release file.
### Security
- Require ADMIN_API_KEY for proxied admin requests and trust X-User-DID only when requests include a matching admin API key, replacing the previous trusted-header configuration.
