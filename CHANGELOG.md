## v0.5.0

v0.5.0 makes typed GraphQL filters more useful when clients already know the records or references they want. You can now batch hydrate records by AT-URI and filter inside common nested lexicon shapes without falling back to broad JSON search.

### For users

- Typed public record queries now include a metadata `uri` filter. Use `where: { uri: { eq: ... } }` for one known AT-URI, or `where: { uri: { in: [...] } }` to fetch a batch. The filter reads the stored record URI, not a `uri` key inside the JSON payload, so it still works when a lexicon has its own field named `uri`.
- Generated nested `where` filters now cover arrays, refs, and unions up to three lexicon path segments deep. This makes common relationship queries possible in GraphQL, such as finding collections whose `items` array contains a strong ref with a matching `itemIdentifier.uri`.
- Array `any` filters keep multiple predicates scoped to the same array item. A query that asks for both `itemIdentifier.uri` and `itemIdentifier.cid` only matches when both values are present on one array element, not split across different elements.
- Hypercert activity queries now include `contributorDid` for contributor matching. It covers inline contributor DIDs, legacy bare DID array entries, and `org.hypercerts.claim.contributorInformation` strong refs by checking the referenced record's `identifier`.

### Notes

- Nested scalar filters are exact-match only: `eq`, `in`, and `isNull`. Substring and range operators such as `contains`, `startsWith`, `gt`, and `lt` are intentionally not exposed for nested fields.
- Nested filters stay within the current record except for the activity-specific `contributorDid` compatibility filter. Use `search` or client-side filtering for deeper traversals or arbitrary JSON matching.

## v0.4.0

v0.4.0 makes Certified profile context easier to fetch, adds a Railway-friendly way to scrape Tap metrics, and fixes data formatting problems in large GraphQL reads and legacy backfills.

### For users

- Public record GraphQL nodes now include `certifiedProfileData`. Clients can fetch the author's Certified profile record and the external labels attached to that profile record alongside any indexed record.
- Large GraphQL pages with external labels now work more reliably on SQLite. Requests for up to 1,000 labeled records no longer exceed SQLite query limits.
- Legacy Go backfills now preserve AT Protocol CBOR values correctly. CID links, byte arrays, and blob references are stored using their canonical JSON shapes.

### For operators

- Hyperindex now includes a small Prometheus Railway service for scraping Tap metrics. Use it when Tap ingestion metrics need to be collected alongside the rest of the Railway deployment.
- Delete activity events now write empty or JSON `null` payloads as `{}`. This avoids PostgreSQL JSONB insert failures for delete events.

### Breaking change and upgrade notes

- The activity log table has been renamed from `jetstream_activity` to `indexing_activity`. The app runs this migration at startup.
- Update any direct SQL queries, dashboards, alerts, backups, or monitoring that still refer to `jetstream_activity`.
- Existing GraphQL queries do not need to change. Clients can opt into `certifiedProfileData` when they need Certified profile context.

## v0.3.0

This release makes Hyperindex easier to operate with external labelers and nicer to use when exploring the GraphQL API.

The biggest change is first-class support for external ATProto labelers. Hyperindex can now subscribe to configured `com.atproto.label.subscribeLabels` streams, keep resumable cursors, store the raw label events, and expose active labels through GraphQL. Clients can also filter generated record queries by external labels before pagination, so paging through labeled records is predictable instead of awkward.

### For users

- GraphiQL now includes the official Explorer plugin. You can build public and admin queries from the sidebar by checking fields instead of typing every query by hand.
- Record queries can filter on the presence of complex top-level lexicon fields.
- Malformed typed union values now resolve to `null` instead of failing the whole GraphQL response.
- GraphQL connection queries can return up to 1000 records per request, up from 100.
- Record-level `externalLabels` now works for CID-scoped records on PostgreSQL.

### For operators

- Admin settings and GraphQL now support removing configured external labeler subscription URLs.
- Labeler ingestion has safer subscription URL validation, log redaction, database indexes, and documentation for running the pipeline with Tap.
- Readiness checks now fail when a configured labeler hits a non-retryable cursor error, with diagnostics that explain what is stuck. `/health` stays process-only, while `/ready` checks the database and configured labeler diagnostics.
- There is an opt-in API smoke write-through check for testing ATProto create, update, and delete ingestion for Certified profiles and Hypercert activity claims.
- Backend Railway builds should be lighter thanks to a smaller Docker build context, the matching Go builder image, and stripped release binary debug metadata.
- Backend status endpoints now report the released Hyperindex version when the binary is built through the Makefile or Docker.
