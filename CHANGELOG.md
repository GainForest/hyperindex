## v0.4.0
### Added
- Add certifiedProfileData to public record GraphQL nodes so clients can fetch an author's Certified profile record and its profile-record external labels alongside any indexed record.
- Add a small Prometheus deployment service for scraping Tap metrics in Railway.
### Breaking
- Rename the activity log table to `indexing_activity`. Operators with direct SQL or monitoring against `jetstream_activity` should update those references; delete activity events now store empty or JSON null payloads as `{}` so PostgreSQL JSONB inserts succeed.
### Fixed
- Fix external label hydration for large GraphQL pages so requests for 1,000 records with labels no longer hit SQLite query limits.
- Fix legacy Go backfill CAR record conversion so AT Protocol CBOR values such as CID links, bytes, and blobs are stored with their canonical JSON shapes.

## v0.3.0

This release makes Hyperindex easier to operate with external labelers and nicer to use when exploring the GraphQL API.

The biggest change is first-class support for external ATProto labelers. Hyperindex can now subscribe to configured `com.atproto.label.subscribeLabels` streams, keep resumable cursors, store the raw label events, and expose active labels through GraphQL. Clients can also filter generated record queries by external labels before pagination, so paging through labeled records is predictable instead of awkward.

### For users

- GraphiQL now includes the official Explorer plugin. You can build public and admin queries from the sidebar by checking fields instead of typing every query by hand.
- Record queries can filter on the presence of complex top-level lexicon fields.
- Malformed typed union values now resolve to `null` instead of failing the whole GraphQL response.
- GraphQL connection queries can return up to 1000 records per request, up from 100.
- Record-level `externalLabels` now works for CID-scoped records on PostgreSQL.

### For operators

- Admin settings and GraphQL now support removing configured external labeler subscription URLs.
- Labeler ingestion has safer subscription URL validation, log redaction, database indexes, and documentation for running the pipeline with Tap.
- Readiness checks now fail when a configured labeler hits a non-retryable cursor error, with diagnostics that explain what is stuck. `/health` stays process-only, while `/ready` checks the database and configured labeler diagnostics.
- There is an opt-in API smoke write-through check for testing ATProto create, update, and delete ingestion for Certified profiles and Hypercert activity claims.
- Backend Railway builds should be lighter thanks to a smaller Docker build context, the matching Go builder image, and stripped release binary debug metadata.
- Backend status endpoints now report the released Hyperindex version when the binary is built through the Makefile or Docker.

## v0.2.0
### Added
- Add a Lexicons page ZIP upload flow so admins can install unpublished lexicon JSON archives without returning to onboarding.
- Add frontend Railway deployment configuration so the client service uses its root page as the health check with a longer startup timeout.
### Breaking
- Rename Hypergoat technical identifiers to Hyperindex, including the Go module path, command entrypoint, binary names, Docker/Compose defaults, and session cookie name.
### Changed
- Route port redirects through the proxy so deployments consistently expose the expected service endpoint.
### Fixed
- Prevent active accounts from being purged during Tap identity processing so indexed records are not removed after harmless identity updates.

## v0.1.0
### Added
- Automate release-note batching so maintainers can generate a reviewable release PR from GitHub Actions on main.
- Add a curated release-note workflow so noteworthy changes can be captured in a consistent, reviewable format.
### Changed
- Change admin access management so ADMIN_DIDS on the backend is the source of truth, removing in-app admin DID mutation paths and showing the admin list as read-only in onboarding and settings.
- Update the release workflow so rerunning it after the release PR is merged creates the version tag and publishes the matching GitHub Release from the generated Changie release file.
### Security
- Require ADMIN_API_KEY for proxied admin requests and trust X-User-DID only when requests include a matching admin API key, replacing the previous trusted-header configuration.
