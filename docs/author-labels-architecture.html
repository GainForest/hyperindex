Issue #92 · GraphQL filter architecture

Labels on the author, not the artifact.

`externalLabels` asks whether the record URI was labeled. `authorLabels` asks whether the record author's DID was labeled. Same label table, same predicate shape, different subject binding.

The split

Record labels target `at://did/collection/rkey` and answer “is this exact record labeled?”
Author labels target `did:...` and answer “is this author account labeled?”
Storage stays shared: both subjects live in `external_label.uri`.

The orglabeler can listen to profile/org records as scoring inputs while emitting labels on the DID.
