## v0.4.0

v0.4.0 makes Certified profile context easier to fetch, adds a Railway-friendly way to scrape Tap metrics, and fixes data formatting problems in large GraphQL reads and legacy backfills.

### For users

- Public record GraphQL nodes now include `certifiedProfileData`. Clients can fetch the author's Certified profile record and the external labels attached to that profile record alongside any indexed record.
- Large GraphQL pages with external labels now work more reliably on SQLite. Requests for up to 1,000 labeled records no longer exceed SQLite query limits.
- Legacy Go backfills now preserve AT Protocol CBOR values correctly. CID links, byte arrays, and blob references are stored using their canonical JSON shapes.

### For operators

- Hyperindex now includes a small Prometheus Railway service for scraping Tap metrics. Use it when Tap ingestion metrics need to be collected alongside the rest of the Railway deployment.
- Delete activity events now write empty or JSON `null` payloads as `{}`. This avoids PostgreSQL JSONB insert failures for delete events.

### Breaking change and upgrade notes

- The activity log table has been renamed from `jetstream_activity` to `indexing_activity`. The app runs this migration at startup.
- Update any direct SQL queries, dashboards, alerts, backups, or monitoring that still refer to `jetstream_activity`.
- Existing GraphQL queries do not need to change. Clients can opt into `certifiedProfileData` when they need Certified profile context.
