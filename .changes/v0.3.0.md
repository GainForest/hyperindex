## v0.3.0

This release makes Hyperindex easier to operate with external labelers and nicer to use when exploring the GraphQL API.

The biggest change is first-class support for external ATProto labelers. Hyperindex can now subscribe to configured `com.atproto.label.subscribeLabels` streams, keep resumable cursors, store the raw label events, and expose active labels through GraphQL. Clients can also filter generated record queries by external labels before pagination, so paging through labeled records is predictable instead of awkward.

### For users

- GraphiQL now includes the official Explorer plugin. You can build public and admin queries from the sidebar by checking fields instead of typing every query by hand.
- Record queries can filter on the presence of complex top-level lexicon fields.
- Malformed typed union values now resolve to `null` instead of failing the whole GraphQL response.
- GraphQL connection queries can return up to 1000 records per request, up from 100.
- Record-level `externalLabels` now works for CID-scoped records on PostgreSQL.

### For operators

- Admin settings and GraphQL now support removing configured external labeler subscription URLs.
- Labeler ingestion has safer subscription URL validation, log redaction, database indexes, and documentation for running the pipeline with Tap.
- Readiness checks now fail when a configured labeler hits a non-retryable cursor error, with diagnostics that explain what is stuck. `/health` stays process-only, while `/ready` checks the database and configured labeler diagnostics.
- There is an opt-in API smoke write-through check for testing ATProto create, update, and delete ingestion for Certified profiles and Hypercert activity claims.
- Backend Railway builds should be lighter thanks to a smaller Docker build context, the matching Go builder image, and stripped release binary debug metadata.
- Backend status endpoints now report the released Hyperindex version when the binary is built through the Makefile or Docker.
