## v0.9.1

v0.9.1 tightens up the new `endorsementClosure` query from v0.9.0. The query now handles messy badge-award data more defensively and reports large endorsement graphs more accurately.

### Fixed

- Endorsement closure results now ignore malformed badge-award subject DIDs instead of treating them as graph edges. In practice, bad or non-account subjects no longer leak into the endorsement path for an account.
- Certified profile hydration stays batched on SQLite, avoiding the large per-DID lookup path that could make endorsement closure responses slower on bigger result sets.
- The `truncated` flag is now set from the actual closure traversal result. If the server cap cuts off the graph, clients can trust `truncated: true`; if invalid edges were skipped and the graph still fits, it stays `false`.
