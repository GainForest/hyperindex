## v0.9.0

v0.9.0 adds a GraphQL query for exploring Certified endorsement networks from a single account. It answers questions like "who does this account endorse?" and "who is reachable through those endorsements?" without making clients download every badge award and rebuild the graph themselves.

### Added

- Added `endorsementClosure(where, first, after)`, a bounded, forward-paginated connection rooted at one DID. Pass `where: { did: { eq: "did:plc:..." } }` to return active account endorsements up to three hops away. Use `where.degree.eq` to limit results to direct endorsements (`1`) or indirect hops (`2` or `3`).
- Returned accounts include `did`, `degree`, optional `certifiedProfileData`, and `viaAccounts` so clients can show the path that led to an indirect account. Results are ordered by degree and DID, include `totalCount`, and set `truncated` when the server cap cuts off part of the graph.
- The resolver only treats active Certified endorsement badge awards for account DID subjects as endorsement edges. It ignores record strong-ref subjects and respects badge-definition `allowedIssuers`, so restricted badge definitions only count endorsements from allowed issuers.

### Notes

- The closure is computed at request time from indexed badge award, badge definition, and badge response records. No separate endorsement edge table or backfill is needed.
- `endorsementClosure` is intentionally rooted at one DID, so the root DID filter only supports `eq`.
